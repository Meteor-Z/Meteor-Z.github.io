# 实习有感

截至到2024年11月15日，我目前在两家公司有段实习，有很多感受，闲来无事，就写了下来

- `沟通`:很多时候，我觉得程序员需要是技术能力强的，自认为我技术能力强（其实很垃圾，在公司都做过臭事），但其实不然，很多时候其实是沟通，就是双方要沟通到位，在第一家公司的时候，我的ld在离职前就对我说，我觉得你的技术是不错的，但是沟通上不是很行，这个你得改改，第二家公司实习的时候，也是因为沟通的问题造成了大锅，所以说，沟通是非常重要的，因为可能沟通不到位，就导致你做的东西其实是错误的，跟实际的有出路。自己是程序员，并且天生不会说话，无法表达，吃了大亏。哎。这个还得练！（
  - 对于我来说，其实很多时间都没有沟通好，导致做错了很多事情，因为我学计算机都是自学，很少问其他人，导致我很少问其他人，甚至我觉得问其他人多了，你觉得你很啰嗦，技术能力很菜。但其实正常跟同事沟通，其实是不错的，这方面吃了不少亏
- `明确目的`：对于项目需求，或者说需求完善等，我认为对于自己，都要设计一个大纲，保证你做的是正确的，并且罗列大纲能够保证你的思路是清晰的，保证不出错，
  - 其实对于我来说，可能是比较冲动，但其实罗列大纲，想一想，能够节约不少时间
- `不要羞涩`: 对于实习离职，我当时很难说出口，但最后我发现，导师，领导，都其实不会说你怎么怎么走了，哎。更多的是希望你最后能找一个更好的，我的两个导师当时也对我说了很多（第二家公司的导师还说了一个多小时，哈哈），大胆说出口就行了，没什么不好意思的，大家都是为了赚钱。
